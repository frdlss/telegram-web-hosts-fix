# 📱 Telegram Web Fix

**Открывает [web.telegram.org](https://web.telegram.org) в браузере, когда провайдер режет IP дата-центров Telegram.**  
Только файл `hosts` на **вашем** компьютере. Без VPN, без прокси, без сервера.

## 🏠 Всё на вашем компьютере

Скачиваете репозиторий, запускаете `telegram-web-fix.bat` у себя на Windows. Меняется только локальный `hosts`, рабочий IP лежит в вашем `telegram-web-ip.cfg`. Серверов, облака и общей инфраструктуры нет — каждый настраивает себя сам.

### На остальную систему это не влияет

Bat-файл и обход через `hosts` **не трогают** глобальные настройки сети:

- **Интернет в целом** и обычные сайты работают как раньше; меняются только адреса из списка `lists/telegram-hosts-domains.txt` (Telegram Web).
- **VPN** — не выключается, не переустанавливается и не подменяется; скрипт к VPN не обращается.
- **Системный прокси, параметры браузера** «на весь интернет» — не меняются.
- **Брандмауэр, таблица маршрутов, DNS-сервер провайдера** — не редактируются (кроме сброса кэша DNS командой после применения `hosts`).
- **Другие программы** (игры, стриминг, офис, **Telegram Desktop**) — этот проект их не затрагивает; касается только **веб-версии** в браузере по указанным доменам.

Иначе говоря: правится один текстовый файл `hosts`, и только нужные строки для Web Telegram — без «глобальных» изменений ОС и без отдельного сервера.

---

## 💡 Когда это имеет смысл

Иногда DNS отдаёт нормальный адрес Telegram, а **порт 443 до этого IP** у провайдера не открывается. Тогда может помочь **другой IP** того же дата-центра (смена DC через `hosts`).

Утилита перебирает список кандидатов, проверяет **TCP 443** и прописывает рабочий IP в `hosts` **только для доменов из** `lists/telegram-hosts-domains.txt`. Остальные сайты и Telegram Desktop не трогает.

| Ситуация | Результат |
|----------|-----------|
| ✅ Режут один IP Telegram, другие ещё живы | Скорее всего поможет |
| ❌ Заблокирован весь Telegram или режут по SNI | Нужен другой способ |
| ❌ Нужен именно Desktop, не браузер | Этот проект не про Desktop |

## 🚀 Как пользоваться

1. Скачайте репозиторий (**Code → Download ZIP** или `git clone`).
2. Запустите **`telegram-web-fix.bat`**: при двойном щелчке **сам запросит UAC** и откроет меню уже с правами администратора (нажмите «Да»). Либо ПКМ → **«Запуск от имени администратора»**. Тексты меню в bat — **на русском**; файл на диске в **UTF-8 без BOM**; в начале выполнения выставляется **`chcp 65001`**, чтобы кириллица и псевдографика рамок отображались нормально.
3. В меню по порядку: **`4`** (найти IP) → **`1`** (включить).
4. Откройте **web.telegram.org** в браузере.

Пункт **`4`** выводит в консоль **диапазон номеров и сами IP каждой «волны»** проверки. Адреса проверяются **параллельно** (по умолчанию до 12 за раз), поэтому полный список проходит **заметно быстрее**, чем по одному IP. При необходимости задайте число потоков (1–32): переменная окружения **`TG_SCAN_PARALLEL`** (например `16`) или параметр `-Parallel` при ручном вызове `telegram-autodetect.ps1`. Полный лог по каждому неудачному IP: **`TG_SCAN_VERBOSE_LOG=1`**.

С очень длинным списком в `lists/telegram-ip-candidates.txt` поиск всё ещё может занять **несколько минут**; можно укоротить файл вручную — в начале идут приоритетные IP.

Если после пункта **4** между строками **несколько секунд тишины** — чаще всего это **таймаут сети** на DNS или на проверку TCP 443 к волне IP; в логе строки **`[dns]`** (четыре резолвера **параллельно**, до ~4 с на всё сразу) и **`[tcp]`** перед волнами. **Пиксельный растр** и «кривые» рамки (`_` / `|`) часто из‑за растрового шрифта в conhost. При старте меню вызывается **`utils/set-console-ttf.ps1`** (Win32 API в **той же** консоли): по очереди пробуются **Consolas**, **Cascadia Mono**, **Lucida Console**. Если шрифт не сменился — **Свойства** окна → **Шрифт** → **Consolas** от 16 pt. Удобно запускать bat из [**Windows Terminal**](https://aka.ms/terminal). Права администратора для `hosts` по-прежнему нужны; **100 колонок** в `mode con`; скан пишет в ту же консоль.

**Первый раз только так:** сначала `4`, потом `1`. До поиска IP в меню не показывается чужой адрес — только после `4` появится **ваш** IP в конфиге. Пункт `1` до этого недоступен по смыслу (скрипт попросит сначала выполнить `4`).

После **1** записи в **файле** `hosts` остаются после перезагрузки ПК, пока вы сами не нажмёте **2**. Отдельная «автозагрузка» в меню не нужна.

### Редактирование `telegram-web-fix.bat`

Собранный **`telegram-web-fix.bat`** — **UTF-8 без BOM** (первая строка `@echo off` остаётся чистым ASCII). Правки текста и рамок вносятся в **`scripts/rebuild_bat_ru.py`** (UTF-8), затем `python scripts/rebuild_bat_ru.py`. В Cursor / VS Code для `.bat` удобно оставить кодировку **UTF-8** (`files.encoding` уже может быть utf8).

## Меню

| | |
|:---:|:---|
| **1** | Включить fix в `hosts` |
| **2** | Выключить — убрать только наши строки |
| **3** | Показать, что сейчас в `hosts` про Telegram |
| **4** | Найти рабочий IP и сохранить в конфиг |
| **0** | Выход |

У bat-файла остаются параметры **enable-silent** и **disable-silent** для автоматизации или ярлыков (без меню).

---

## После перезагрузки

Fix **останется включённым**, если строки в `hosts` никто не удалил (антивирус, другой софт, очистка).

Если перестало открываться через время — провайдер мог заблокировать и этот IP. Снова **4**, потом **1**.

---

## Как выключить

1. **2** в меню — снять записи из `hosts`  
2. Папку с проектом можно просто удалить (сначала лучше **2**)

## 🩹 Если не заработало

- **Окно cmd сразу закрывается / `'...' is not recognized`** — пересоберите bat: `python scripts/rebuild_bat_ru.py`. Не правьте `.bat` в редакторе так, чтобы появился **UTF-8 BOM** перед `@echo off`, не смешивайте кодировки и не сохраняйте файл в **CP1251** (псевдографика там недоступна). Нужен **Windows 10** не старее **1903** / **Windows 11** для корректного разбора UTF-8 bat; при «кракозябрах» проверьте, что скрипт доходит до `chcp 65001`, и шрифт консоли с кириллицей (**Consolas** и т.п.).
- **Пиксельный шрифт, «ломаная» рамка** — при старте и перед **[4]** вызывается **`utils/set-console-ttf.ps1`** (переключение на Consolas / Cascadia / Lucida через Win32). Если не помогло: **Свойства** окна → **Шрифт** → **Consolas** от 16; либо [Windows Terminal](https://aka.ms/terminal). Вручную проверить скрипт: `powershell -NoProfile -File utils\set-console-ttf.ps1` из той же открытой консоли.
- **В меню «IP не найден» после [4]** — до ваших серверов Telegram на **443** не достучаться ни из списка, ни с разных DNS; такое встречается при сильной региональной блокировке. Имеет смысл другая сеть (мобильный интернет / раздача), VPN или ручное дополнение `lists/telegram-ip-candidates.txt` из [cidr.txt](https://core.telegram.org/resources/cidr.txt). Этот проект **не заменяет VPN**, если режут все подсети. Перед текстом ошибки скрипт выводит **сводку** (сколько IP проверено, сколько DNS ответили, был ли второй проход проверки) — по ней проще понять, не «сломался ли» поиск, а просто **нет живого маршрута до DC**.
- **Сайт не открывается** — снова **4 → 1**, в cmd выполните `ipconfig /flushdns`
- **Не грузится QR для входа** — убедитесь, что fix включён (**1**); если в DevTools (F12) виден новый домен — напишите в Issues
- **После обновления Windows** — при необходимости снова **1** или **4 → 1**, `ipconfig /flushdns`
- **Вчера работало, сегодня нет** — снова **4**, подобрать другой IP
- **Нужен Desktop** — здесь только Web в браузере  

Список IP иногда устаревает. Актуальные подсети Telegram: [core.telegram.org/resources/cidr.txt](https://core.telegram.org/resources/cidr.txt) — при необходимости допишите адреса в `lists/telegram-ip-candidates.txt`.

## Что трогает программа

- Только `C:\Windows\System32\drivers\etc\hosts` — строго по списку доменов в `lists/telegram-hosts-domains.txt` (не «весь интернет»).
- Наши строки в `hosts` сгруппированы под одним служебным комментарием-маркером — **2** удаляет только этот блок.
- При первом включении делается копия `hosts` в папку `backup`.

Остальной трафик, VPN и приложения — см. блок **«На остальную систему это не влияет»** выше.
