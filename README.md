# 📱 Telegram Web Fix

**Открывает [web.telegram.org](https://web.telegram.org) в браузере, когда провайдер режет IP дата-центров Telegram.**  
Только файл `hosts` на **вашем** компьютере. Без VPN, без прокси, без сервера.

## 🏠 Всё на вашем компьютере

Скачиваете репозиторий, запускаете `telegram-web-fix.bat` у себя на Windows. Меняется только локальный `hosts`, рабочий IP лежит в вашем `telegram-web-ip.cfg`. Серверов, облака и общей инфраструктуры нет — каждый настраивает себя сам.

---

## 💡 Когда это имеет смысл

Иногда DNS отдаёт нормальный адрес Telegram, а **порт 443 до этого IP** у провайдера не открывается. Тогда zapret и похожие обходы DPI часто бесполезны — нужен **другой IP** того же дата-центра.

Утилита перебирает список кандидатов, проверяет **TCP 443** и прописывает рабочий IP в `hosts` **только для доменов Web-версии** (около 10 штук). Остальные сайты и Telegram Desktop не трогает.

| Ситуация | Результат |
|----------|-----------|
| ✅ Режут один IP Telegram, другие ещё живы | Скорее всего поможет |
| ❌ Заблокирован весь Telegram или режут по SNI | Нужен другой способ |
| ❌ Нужен именно Desktop, не браузер | Этот проект не про Desktop |

## Что нужно

- Windows 10 или 11  
- Права администратора (без них `hosts` не переписать)  
- Браузер с WebSocket — Chrome, Edge, Firefox  

---

## 🚀 Как пользоваться

1. Скачайте репозиторий (**Code → Download ZIP** или `git clone`).
2. Запустите **`telegram-web-fix.bat`**, в UAC нажмите **Да**.
3. В меню по порядку: **`4`** (найти IP) → **`1`** (включить).
4. Откройте **web.telegram.org** в браузере.

**Первый раз только так:** сначала `4`, потом `1`. Пункт `4` сам `hosts` не меняет — только ищет IP и пишет его в `telegram-web-ip.cfg`.

---

## Меню

| | |
|:---:|:---|
| **1** | Включить fix в `hosts` |
| **2** | Выключить — убрать только наши строки |
| **3** | Показать, что сейчас в `hosts` про Telegram |
| **4** | Найти рабочий IP и сохранить в конфиг |
| **5** | Автозапуск при входе в Windows (вкл / выкл) |
| **0** | Выход |

Для Планировщика заданий есть тихий запуск: параметры **autostart**, **enable-silent**, **disable-silent** у того же bat-файла.

---

## Автозапуск

Пункт **5** создаёт задачу **TelegramWebFix**: после входа в систему снова подставляется IP из конфига, без повторного скана.

Если перестало открываться через время — провайдер мог заблокировать и этот IP. Снова **4**, потом **1**.

---

## Как выключить

1. **2** в меню — снять записи из `hosts`  
2. **5** — отключить автозапуск, если включали  
3. Папку с проектом можно просто удалить  

---

## 🩹 Если не заработало

- **Сайт не открывается** — снова **4 → 1**, в cmd выполните `ipconfig /flushdns`
- **Не грузится QR для входа** — убедитесь, что fix включён (**1**); если в DevTools (F12) виден новый домен — напишите в Issues
- **После обновления Windows** — снова **1** или перелогин с включённым **5**
- **Вчера работало, сегодня нет** — снова **4**, подобрать другой IP
- **Нужен Desktop** — здесь только Web в браузере  

Список IP иногда устаревает. Актуальные подсети Telegram: [core.telegram.org/resources/cidr.txt](https://core.telegram.org/resources/cidr.txt) — при необходимости допишите адреса в `lists/telegram-ip-candidates.txt`.

## Что трогает программа

- Только `C:\Windows\System32\drivers\etc\hosts`
- Наши строки помечены `# zapret-telegram-web-fix` — **2** удаляет только их
- При первом включении делается копия `hosts` в папку `backup`

## Дисклеймер

Проект **не связан** с Telegram FZ-LLC. Распространяется как есть, без гарантий.  
Использование `hosts` для обхода ограничений — на вашей ответственности и в рамках законов вашей страны.
